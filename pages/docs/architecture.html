Guest-First Runtime Model

Baskit routes storage and sync behavior based on authentication state.

Mode 1: Guest

  • No sign-in required.
  • Data is stored in local Hive storage.
  • Works offline by default.

Mode 2: Authenticated

  • Google Sign-In with Firebase Auth.
  • Lists and items stored in Firestore with collaboration support.
  • Cross-device access and near real-time updates.

Migration Flow

  1. User starts in guest mode and creates local lists.
  2. User signs in via Google.
  3. App migrates local state to cloud-backed storage.
  4. Future writes route to Firestore-backed repository.

Key Modules

  • app/lib/services/storage_service.dart - storage routing entrypoint
  • app/lib/services/local_storage_service.dart - local persistence
  • app/lib/services/firestore_service.dart and firestore_layer.dart - cloud access
  • app/lib/repositories/storage_shopping_repository.dart - repository abstraction over storage mode
