# Baskit PRD - Google Play Production Readiness

This checklist covers the non-code and release-gating work required before applying for Google Play production access.

## Code Gates
- `flutter analyze` passes on the release branch.
- `flutter test` passes on the release branch.
- Android release artifacts are signed with the release key, not debug signing.
- Firestore rules and indexes are deployed to the production Firebase project before Play review.
- The Play Store release uses an `.aab` built from CI or a locally reproducible release build.

## Firebase And Sign-In
- Add the Google Play App Signing SHA-1 and SHA-256 certificates to the Firebase Android app.
- Keep the upload-key SHA-1 and SHA-256 in Firebase if local or CI signed builds are used for testing.
- Download and commit an updated `google-services.json` after Firebase certificate changes.
- Verify Google Sign-In from a Play-distributed internal or closed testing build.
- Verify guest-to-Google migration from a Play-distributed build with existing local lists.

## Play Console Setup
- Complete App Content forms:
  - Privacy Policy URL: `https://kmjayadeep.github.io/baskit/privacy-policy.html`
  - Account deletion URL: `https://kmjayadeep.github.io/baskit/delete-account.html`
  - Data Safety answers aligned with the current privacy policy.
  - Target audience and content rating.
- Confirm the Data Safety form does not claim analytics or crash reporting unless those SDKs are added.
- Upload release notes matching the shipped version.
- Upload the release `.aab` and native debug symbols generated by CI.

## Closed Testing Evidence
- Run the required closed test before applying for production access.
- Maintain at least 12 opted-in testers for at least 14 days if required for the developer account.
- Give testers a script that covers:
  - Guest list creation, editing, deletion, and offline usage.
  - Google Sign-In.
  - Local-to-cloud migration after sign-in.
  - Sharing by email.
  - Leave shared list.
  - Owner removes member.
  - Sign-out and return to guest mode.
  - Manual account deletion request path through the public web page.
- Collect concrete feedback, issue links, or screenshots from testers for the production-access application.

## Manual Account Deletion Process
- Monitor `contact@jayadeep.me` for deletion requests.
- Verify the requester controls the email address linked to the Baskit account.
- Delete the user's Firebase Auth account if it still exists.
- Delete `/users/{userId}`.
- Delete lists owned by the user unless the user requested export or transfer first.
- Remove the user from shared lists they do not own.
- Confirm completion by email within 30 days.

## Final Release Smoke Test
- Install the Play-distributed build on a fresh device.
- Create a guest list and item.
- Sign in with Google and confirm the guest list migrates.
- Share a list with a second test account.
- Confirm both accounts see real-time list and item updates.
- Confirm leave-list and remove-member flows revoke access.
- Confirm the About dialog opens the privacy policy.
